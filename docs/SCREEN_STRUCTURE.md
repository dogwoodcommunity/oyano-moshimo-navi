# 画面構成 v0.3

## Web入口: Next.js

### `/`
LP。コピー: 「親の状況が変わったら、家族が次にやることが分かる。」
CTA: 「親の状態から始める」

### `/start`
親の現在ステータスを選択。
- 元気・準備中
- 長期入院
- 施設入居
- 認知症・判断能力低下
- 危篤・看取り準備
- 死亡直後
- 葬儀後
- 相続手続き中
- 実家整理中

ログイン不要。選択後に匿名case作成。

### `/diagnosis`
5分診断。
- 親の状況
- 家族構成
- 不動産/実家の有無
- 銀行/保険/年金の把握状況
- 困っていること
- 家じまい状況
- 写真アップロード任意
- 連絡先任意

### `/result/[caseId]`
診断結果。
- 状況タイプ
- まずやること3つ
- 期限のあるタスク
- 家族に確認すること
- 登録しておく情報
- 必要な相談先カテゴリ
- アプリ引き継ぎCTA
- 発動サポートパックCTA（Web決済）

### `/support-pack`
発動サポートパック説明・Stripe Checkout開始。
アプリ内課金回避のため、Webで完結する人的サポート商品として表現。

### `/pricing`
Free / Family Plus / 発動サポートパック説明。

## 継続アプリ: Expo

### `/(auth)/welcome`
Magic Link / Web診断引き継ぎ。

### `/(tabs)/dashboard`
親ごとの状態と未完了タスク。

### `/people/[id]`
対象者トップ。
- 現在ステータス
- 今日やること
- 家族メンバー
- 未登録情報
- 家じまい進捗

### `/people/[id]/status`
ステータス変更。変更時にstatus_events作成、task_templatesからtasks生成。

### `/people/[id]/tasks`
家族タスクボード。
- 未着手
- 進行中
- 完了
- 期限切れ
- 担当者

### `/people/[id]/assets`
情報の存在・保管場所登録。
暗証番号・パスワードは保存禁止。

### `/people/[id]/timeline`
状況記録。
- 入院した
- 施設入居した
- 葬儀が終わった
- 書類を見つけた
- 業者に相談した

### `/people/[id]/home`
実家カルテ。
- 物件種別
- 空き家状況
- 鍵/ライフライン
- 家財量
- 写真
- 売却/管理/解体/未定

### `/people/[id]/family`
家族招待、権限管理。

### `/notifications`
通知設定。

### `/account/plan`
プラン確認。IAPまたはWeb購入済み状態の表示。
