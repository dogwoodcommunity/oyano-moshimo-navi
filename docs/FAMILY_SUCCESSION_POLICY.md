# 家族ボード承継方針

親のもしもナビは「低頻度・高重要度」のアプリなので、家族代表ひとりだけに運用権限が集中すると、代表者が入院・死亡・端末紛失・連絡不能になった時に家族ボードが止まる。

## v0.3の方針

- 家族代表または管理者は、既存メンバーを共同管理者にできる。
- 共同管理者は `family_members.role = 'owner'` として扱う。
- 直接「owner招待」は作らない。まず家族として招待し、信頼できる相手だけ後から昇格する。
- 代表者を降格する操作はMVPでは作らない。誤操作で誰も管理できなくなるリスクを避けるため。
- `families.owner_user_id` はprimary ownerとして残す。共同管理者を増やしても無料招待枠を増やさないため、Freeの人数判定は `owner_user_id` 以外を数える。

## できること

- 代表者が使えなくなった時も、共同管理者が家族ボード、担当者、期限管理を継続できる。
- 未割当タスクや期限通知の送信対象にも、共同管理者が含まれる。

## まだ作らないこと

- オーナー自身の死亡届出フロー
- 法的な相続・代理権限の判定
- 家族内の権限争いを解決する審査機能
- 自動的な代表者交代

この領域は法務・運用の論点が大きいため、v0.3では「信頼できる家族を事前に共同管理者にする」までに留める。
